为了避免给服务器造成过大压力，造成资源浪费，我们可以将一些 API 数据缓存在客户端本机中。这种缓存的对象应该具有以下特点：

  1. 数据量少。
  2. 及时性要求不高，不需要实时更新。
  3. 数据较为固定。

对于缓存 API 的方案，分为两种。一是单独封装工具供开发者调用，二是在 http 请求工具中使用缓存代理
